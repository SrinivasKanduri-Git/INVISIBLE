# openapi-generator

Produces or refreshes the OpenAPI 3.1 description of HTTP endpoints. Reads code as truth; treats existing spec as a target to reconcile. Pairs with [[api-net]] (API docs discipline is enforced inline at write-time; openapi-generator runs the bulk job).

## When to run

- Need API docs for a new service (no spec exists).
- Existing spec drifted from code — reconcile.
- Pre-launch on a public/partner API.
- Onboarding consumers — give them an accurate spec.
- After major endpoint refactor.

## When NOT to run

- Tiny project with ≤5 endpoints and no external consumers.
- Spec is hand-curated and serves as source of truth (this generator goes code→spec; respect human-curated specs).

## Modes

| Mode | Behavior |
|---|---|
| `generate` | No existing spec; produce one from scratch |
| `refresh` (default) | Existing spec; update with code reality; flag user-edited sections for review |
| `drift-check` | Read-only; report diffs between code and spec; suggest reconciliation |

## How it walks

1. **Discover routes**. Per stack:
   - Rails: `Rails.application.routes.routes`
   - Django: `urls.py` traversal + DRF view introspection
   - Express / NestJS: route registration scan
   - FastAPI: introspect `app` (already OpenAPI-native — refresh = export + tidy)
   - Phoenix: router scope walk
2. **Per route, derive**:
   - Method + path + path/query params
   - Request body shape (from validator / Pydantic / ActiveRecord param permit / DTO / Zod)
   - Response shape (from serializer / response_model / explicit typing / inferred from return)
   - Auth + scopes (from middleware / decorator / guard)
   - Status codes (from raises / throws / explicit returns)
   - Rate-limit annotations (if present)
3. **Schema inference**:
   - From type annotations (TypeScript, Python, Ruby Sorbet/RBS) where available.
   - From DB model (with caveat: model ≠ wire shape; serializer preferred).
   - From validator (Zod, Joi, Yup, Pydantic, dry-validation, JSON Schema annotations).
4. **Example synthesis**:
   - From fixtures / factories / seed data when accessible.
   - Otherwise generate plausible from schema (`faker`-style).
5. **Diff existing spec** (refresh / drift-check):
   - Endpoints in code, not in spec → propose ADD.
   - Endpoints in spec, not in code → propose REMOVE (or mark deprecated).
   - Shape mismatch → propose UPDATE; preserve hand-written descriptions.
6. **Output**:
   - `openapi.yaml` (or `.json`) — full spec.
   - `openapi-drift.md` — human-readable diff report (refresh / drift-check modes).
   - `openapi-todo.md` — endpoints lacking sufficient annotation to spec well.

  description: |

    Source: <repo>@<commit>.

  - url: https://api.example.com
    description: production
  - url: https://staging.example.com
    description: staging
security:
  - bearerAuth: []
paths:
  /orders:
    get:
      summary: List orders for the current tenant
      operationId: listOrders
      tags: [orders]
      parameters:
        - name: limit
          in: query
          schema: { type: integer, minimum: 1, maximum: 100, default: 20 }
        - name: cursor
          in: query
          schema: { type: string }
      responses:
        '200':
          description: OK
          content:
            application/json:
              schema: { $ref: '#/components/schemas/OrderListResponse' }
        '401': { $ref: '#/components/responses/Unauthorized' }
        '429': { $ref: '#/components/responses/RateLimited' }
    post:
      operationId: createOrder
      ...
components:
  securitySchemes:
    bearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
  responses:
    Unauthorized:
      description: Missing/invalid auth
      content:
        application/json:
          schema: { $ref: '#/components/schemas/Error' }
    RateLimited:
      description: Too many requests
      headers:
        Retry-After: { schema: { type: integer } }
      content:
        application/json:
          schema: { $ref: '#/components/schemas/Error' }

      type: object
      required: [items, next_cursor]

        items:
          type: array

          type: [string, "null"]
    Order:
      type: object
      ...
    Error:
      type: object
      required: [code, message]
      properties:
        code: { type: string }
        message: { type: string }
        request_id: { type: string }

## Drift report shape (refresh / drift-check)

```markdown
# OpenAPI Drift Report — <date>

## Summary
- Added endpoints: <N>
- Removed endpoints: <N>
- Modified shapes: <N>
- Spec-only (orphaned): <N>

## Endpoints in code but missing from spec
- `POST /orders/{id}/refund` (added <commit>)
  - Suggested operationId: refundOrder
  - Suggested response shape: <inferred>

## Endpoints in spec but missing in code
- `GET /legacy/v1/items`
  - Last seen in code at <commit> (deleted <date>)
  - Suggest: mark deprecated → sunset, OR remove from spec

## Shape mismatches
- `POST /orders` request body
  - Code: requires `idempotency_key: string`
  - Spec: not mentioned
  - Suggest: add to spec

- `GET /users/{id}` response 200
  - Code returns: `email: string | null`
  - Spec says: `email: string`
  - Suggest: spec → nullable

## Auth annotation gaps
- `POST /webhooks/stripe`: spec marks `security: [bearerAuth]`; code has signature verify, no bearer
  - Suggest: distinct securityScheme `stripeSignature`

## Status codes undocumented
- `POST /orders` raises `409 IdempotencyConflict` but spec lists only `200, 400`
  - Suggest: add `409` to spec
```

## Inference rules

### Schema inference precedence
1. Explicit OpenAPI annotation in code (JSDoc `@openapi`, FastAPI `response_model`, Rails RSwag annotations).
2. Validator / DTO schema (Zod, Pydantic, Joi).
3. Type annotation (TypeScript types, RBS, Sorbet).
4. ORM model (with serializer applied if detected).
5. Best-effort from response examples in tests.

If none yield a schema → mark endpoint in `openapi-todo.md` and produce a placeholder schema with `description: "Schema not inferable — manual annotation needed"`.

### Auth inference
- Middleware chain on route → security scheme.
- Public routes detected by absence of auth middleware → security: `[]` explicitly (don't omit; absence is ambiguous).
- Webhook receivers with signature verify → distinct securityScheme.

### Versioning detection
- URL versioning (`/v1/`, `/v2/`) → multiple `servers` or operations tagged per version.
- Header versioning → `parameters` with header on every op.
- Stripe-style date versioning → custom security scheme + `info.x-api-versions`.

## What it preserves

Refresh mode preserves human edits:
- `description` fields on operations and schemas (unless code annotation supersedes).
- `tags`, `externalDocs`, `examples` added by hand.
- Reordering of paths/schemas (alphabetical default; respect existing order).

Generator never strips fields with extension prefix `x-*` — those are intentional.

## Drift policy

After refresh, three states per endpoint:

| State | Action |
|---|---|
| In sync | No change |
| Drift, code-canonical | Update spec; preserve descriptions |
| Drift, spec-canonical (e.g., spec is the contract before code) | Flag for human review; don't auto-overwrite |

User can configure via CLAUDE.md whether code or spec is canonical (default: code).

## GraphQL adjunct

If GraphQL detected, runs a parallel pass:
- Validate SDL against introspected schema.
- Flag fields in code not in published SDL (and vice versa).
- Cross-references [[graphql-net]] for deprecation discipline.

## Token budget

| Scope | Tokens |
|---|---|
| ≤30 endpoints | 10–25k |
| 30–100 endpoints | 25–60k |
| 100+ endpoints | run per-tag or per-router |

## Failure modes

- Routes can't be discovered (custom dispatcher, dynamic registration) → emit "discovery incomplete" with manual entry-point hint.
- Schemas can't be inferred (untyped vanilla JS / Ruby with no DTO) → produce TODO list; recommend introducing Zod / dry-validation / similar.
- Spec exists in non-standard format → convert to OpenAPI 3.1; flag deviations.

## CLAUDE.md hooks

Reads section A (api_style, api_versioning, auth_scheme), B (project conventions — error envelope, pagination), C (deprecated endpoints kept for compat).
Reads section D (cached patterns).
Writes `openapi.yaml` + drift report.
